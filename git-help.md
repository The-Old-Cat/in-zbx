# после клонирования

## Начало

```sh
git config --global user.name "Ваше Имя"
git config --global user.email "ваш@email.com"
git config --global user.name "Ваше Имя"
git config --global user.password "ваш пароль"
```

Вот минимальный набор команд Git, который позволит вам эффективно работать с репозиториями:

---

### **1. Начало работы**

- **Инициализация нового репозитория**:

  ```bash
  git init
  ```

  Создаёт новый локальный репозиторий в текущей директории.

- **Клонирование существующего репозитория**:

  ```bash
  git clone <url>
  ```

  Копирует удалённый репозиторий на ваш компьютер.

---

### **2. Проверка состояния**

- **Просмотр статуса**:

  ```bash
  git status
  ```

  Показывает текущее состояние репозитория: какие файлы изменены, добавлены или ожидают коммита.

- **Просмотр изменений**:

  ```bash
  git diff
  ```

  Показывает различия между рабочей директорией и индексом (staging area).

---

### **3. Работа с изменениями**

- **Добавление изменений в индекс**:

  ```bash
  git add <file>
  ```

  Добавляет конкретный файл в индекс. Или:

  ```bash
  git add .
  ```

  Добавляет все изменения из текущей директории.

- **Создание коммита**:

  ```bash
  git commit -m "Описание изменений"
  ```

  Фиксирует изменения в истории репозитория.

---

### **4. Работа с ветками**

- **Просмотр списка веток**:

  ```bash
  git branch
  ```

  Показывает все локальные ветки. Текущая ветка отмечена `*`.

- **Создание новой ветки**:

  ```bash
  git branch <branch_name>
  ```

- **Переключение между ветками**:

  ```bash
  git checkout <branch_name>
  ```

  Переключается на указанную ветку. Или:

  ```bash
  git switch <branch_name>
  ```

  (Новая команда в Git 2.23+).

- **Создание и переключение на новую ветку**:

  ```bash
  git checkout -b <branch_name>
  ```

  Или:

  ```bash
  git switch -c <branch_name>
  ```

---

### **5. Работа с удалённым репозиторием**

- **Добавление удалённого репозитория**:

  ```bash
  git remote add origin <url>
  ```

- **Отправка изменений на сервер**:

  ```bash
  git push origin <branch_name>
  ```

  Отправляет локальные изменения в удалённый репозиторий.

- **Получение изменений с сервера**:

  ```bash
  git pull origin <branch_name>
  ```

  Скачивает изменения с удалённого репозитория и объединяет их с локальной веткой.

---

### **6. Просмотр истории**

- **Просмотр журнала коммитов**:

  ```bash
  git log
  ```

  Показывает историю коммитов. Для упрощённого вывода:

  ```bash
  git log --oneline
  ```

---

### **7. Отмена изменений**

- **Отмена изменений в рабочей директории**:

  ```bash
  git checkout -- <file>
  ```

  Возвращает файл к состоянию последнего коммита.

- **Отмена коммита**:

  ```bash
  git reset --soft HEAD~1
  ```

  Отменяет последний коммит, но оставляет изменения в индексе.

  Если нужно полностью удалить последний коммит:

  ```bash
  git reset --hard HEAD~1
  ```

---

### **8. Дополнительные полезные команды**

- **Удаление файла**:

  ```bash
  git rm <file>
  ```

  Удаляет файл из репозитория и рабочей директории.

- **Переименование файла**:

  ```bash
  git mv <old_name> <new_name>
  ```

- **Просмотр настроек**:

  ```bash
  git config --list
  ```
